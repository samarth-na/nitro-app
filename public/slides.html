Architecture Overview

  • Nitro serves HTTP routes & middleware
  • Drizzle ORM typesafe SQL queries
  • better-sqlite3 persistent local DB
  • bcrypt password hashing (10 rounds)
  • jsonwebtoken stateless auth tokens

Database Schema

Single users table — auto-created on first server start.

CREATE TABLE users ( id INTEGER PRIMARY KEY AUTOINCREMENT, email TEXT NOT NULL UNIQUE, password TEXT NOT NULL, created_at TEXT NOT NULL DEFAULT (datetime('now')) ); // Drizzle schema (db/schema.ts) export const users = sqliteTable("users", { id: integer("id").primaryKey({ autoIncrement: true }), email: text("email").notNull().unique(), password: text("password").notNull(), createdAt: text("created_at").notNull().default("CURRENT_TIMESTAMP"), });

No manual migration needed — db.ts runs CREATE TABLE IF NOT EXISTS on import.

API Endpoints

POST /api/auth/signup

Create a new user account.

{ "email": "you@example.com", "password": "securePass123" }
// 201 Created { "message": "User created successfully", "user": { "id": 1, "email": "you@example.com" }, "token": "eyJhbG..." }

POST /api/auth/signin

Authenticate and receive a token.

{ "email": "you@example.com", "password": "securePass123" }
// 200 OK { "message": "Sign in successful", "user": { "id": 1, "email": "you@example.com" }, "token": "eyJhbG..." }

Token expires in 24h. Set JWT_SECRET env var in production.

Route Protection

Middleware middleware/auth.ts is auto-detected by Nitro and guards every request.

const PUBLIC_ROUTES = ["/api/auth/signin", "/api/auth/signup"]; export default defineEventHandler(async (event) => { if (PUBLIC_ROUTES.includes(url.pathname)) return; // pass through const authHeader = event.headers.get("authorization"); if (!authHeader?.startsWith("Bearer ")) throw createError({ statusCode: 401, statusMessage: "Unauthorized" }); const token = authHeader.slice(7); const payload = jwt.verify(token, JWT_SECRET); event.context.auth = { user: payload }; });

Public

Signin & signup routes skip JWT verification entirely.

Protected

All other routes require a valid Bearer token in the Authorization header.

Context

Valid tokens attach user info to event.context.auth for downstream handlers.

Input Validation

Email

Must match /^[^\s@]+@[^\s@]+\.[^\s@]+$/

400 Bad Request — "Invalid email format"

Password

Minimum 8 characters

400 Bad Request — "Password must be at least 8 characters"

Required Fields

Both email and password must be present

400 Bad Request — "Email and password are required"

Duplicate Email

SQLite UNIQUE constraint blocks duplicates

409 Conflict — "User already exists"

How to Use as a Backend

1. Start the Server

bun run dev # http://localhost:3000

2. Sign Up

curl -X POST http://localhost:3000/api/auth/signup \\ -H "Content-Type: application/json" \\ -d '{"email":"a@b.com","password":"pass1234"}'

3. Sign In

curl -X POST http://localhost:3000/api/auth/signin \\ -H "Content-Type: application/json" \\ -d '{"email":"a@b.com","password":"pass1234"}'

4. Call Protected Routes

# Save the token from signin response export TOKEN="eyJhbG..." curl http://localhost:3000/api/example \\ -H "Authorization: Bearer $TOKEN"

5. Production Build

JWT_SECRET="your-secret" bun run build # Preview the production build npx srvx --prod .output/
